# Лабораторная работа 2 (medium) 

## Запуск программы

Через терминал командой `python -m src.main`

## Поддерживаемые команды

1. **ls** с поддержкой подробного вывода -l
2. **cd**
3. **cat** может выводить несколько подряд указанных файлов, при указании несуществующего файла, выведется ошибка, а затем остальные файлы
4. **cp** копирование файлов/каталогов, копирование несокльких файлов в каталог, поддерживается -r, для рекурсивного копирования
5. **mv** перемещение/переименование файла/каталога, можно переместить несколько файлов в папку
6. **rm** удаление указанного файла, при удалении каталога запрашивается разрешение, поддержание -r для удаления каталога со всем содержимым, запрет на удаление родительской и корневой дирректориии
7. **zip/tar** создание архива указанного формата из каталога
8. **unzip/untar** распаковка архива указанного формата
9. **grep** поиск совпадений по паттерну, поддерживается -r -i для рекурсивного поиска в папках и для игнорировния регистра при поиске
10. **history** вывод последних n клманд, которые хранятся в файле src/.history, при неуказанном n будет выведено 10 последних команд (в истории сохраняются 10 последних)
11. **undo** для команд cp mv и rm. Для undo rm вместо удаления перемещаем файлы/каталоги в папку commands/.trash

## Формат ввода

Строка, команда с аргументами

## Принятые решения
1. Аргументы обрабатывабтся при помощи библиотеки shlex
2. Все команды расположены в отдельном каталоге commands, .history, shell.log и .history  также находятся в этом каталоге
3. Обработка команд происходит в mini_shell, в main только запуск оболочки

## Алгоритм работы
1. Пользователь вводит команду
2. Команда записывается в .history, логируется и парсится
3. Выполняется соответствующая функция из commands
4. Результат выводится в консоль, в случае ошибки она также выводится в консоль и логируется
5. Ошибки логируются в соответсвующих функциях команд и записываются в shell.log, то есть сначала логируется введенная команда, а затем, в случае ошибки, записывается выведенная ошибка
